# TTS 并行加速器

> 基于 GPT-SoVITS，支持双机并行合成 + 自动故障切换。多句文本自动拆分后分发给多台 TTS 服务器同时合成，朗读速度比单机提升 **1.5~2x**。
>
> 支持作为**代理服务器**部署，让局域网内任意机器的 TTS 客户端都能透明享受加速，兼容第三方 GPT-SoVITS 客户端（GET/POST 均支持）。

## 1. 快速开始（5 分钟上手）

### 环境要求

- Python 3.10+（标准库，**无需安装额外依赖**）
- Windows（播放用 `winsound`）或 Linux（用 `aplay` / `paplay`）
- 能访问局域网内的 GPT-SoVITS TTS 服务器（端口 9880）

### 最简调用（命令行）

```bash
python tts_reply.py 可莉 "你好！今天天气真不错，我们一起去冒险吧！"
```

运行后会自动：分句 → 并行合成 → 合并 WAV → 播放。

### 启动图形配置界面

```bash
python tts_gui.py
# 或双击 启动配置器.bat
```

### 仅启动代理服务（无 GUI）

```bash
python tts_proxy.py --port 9882
# 或双击 启动代理服务.bat
```

### 检查服务器是否在线

```bash
python check_tts_servers.py
```

## 3. 三种运行模式

在 GUI「全局设置」标签页切换，也可在 `tts_gui_config.json` 中修改 `"mode"` 字段。

### 🖥️ 代理服务器模式（推荐，多机部署）

```
[客户端机器A] tts_reply.py → http://代理机IP:9882/tts
[客户端机器B] 第三方TTS客户端 → http://代理机IP:9882/tts
                                        ↓
                              [代理机] tts_proxy.py
                                   /          \
                           [TTS服务器1]    [TTS服务器2]
```

- 本机启动代理（默认端口 9882），对外暴露统一接口
- 收到请求后自动拆句、并发调度多台 TTS 服务器
- 客户端无需关心服务器列表，只填代理地址即可

### 💻 客户端模式

本机不直连 TTS 服务器，所有请求转发给远程代理机。适合多台 PC 共用一台代理的场景。

在「全局设置」填写代理机的地址，例如 `http://192.168.28.11:9882/tts`。

### ⚡ 直连模式（兼容旧版）

`tts_reply.py` 自行管理并发，直接连接配置的服务器组。单机使用或测试时方便，多机部署时加速效果会打折。

## 4. 图形界面（GUI）配置工具

启动：`python tts_gui.py` 或双击 `启动配置器.bat`

GUI 包含 5 个标签页：

- 切换运行模式（代理服务器 / 客户端 / 直连）
- 设置音频输出目录
- 设置并行触发阈值（句数 ≥ N 时启用并行，建议 2）

### 🎭 角色管理

增删改角色音色配置：
- 角色名称
- 参考音频路径（TTS 服务器上的路径）
- 参考音频台词
- 参考音频语言 / 合成文本语言

### 🖥️ 服务器管理

- 按物理机器分组管理所有 TTS 服务器 IP
- 可视化树形列表，支持拖拽排序、批量启用/禁用
- 一键检测所有 IP 的在线状态
- 实时生效：服务器启用状态变更后自动热重载，无需重启代理

### 🔀 代理服务

- 启动 / 停止代理服务
- 配置互联通讯端口（默认 9882）
- 可选开启本机 TTS 监听端口（默认 9880，供 tts_reply.py 本地使用）
- 自动启动选项
- 调试模式开关（见[调试模式](#10-调试模式)）
- 实时日志 + 请求统计

### ▶️ 测试播放

- 选角色、填台词，一键测试合成效果
- 实时日志显示分句、并发分配、耗时、后端使用情况

## 5. 代理服务（tts_proxy.py）

### 兼容性

代理服务完全兼容 GPT-SoVITS 原生 `/tts` 接口，**任何能调用 GPT-SoVITS 的客户端，把地址改成代理地址即可**，无需修改客户端代码。

- ✅ GET 请求（URL 参数）
- ✅ POST 请求（JSON Body）
- ✅ POST 请求（application/x-www-form-urlencoded）
- ✅ 自动修正 GET 参数类型（字符串 → int/float/bool/list）
- ✅ 自动清理文本中的 HTML 标签（`<br>`、`<p>` 等不会被朗读）

### API 端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/tts` | GET / POST | TTS 合成（主接口） |
| `/status` | GET | 代理状态 JSON |
| `/backends` | GET | 后端列表 JSON |
| `/health` | GET | 健康检查 |

### 角色字段自动补全

第三方客户端若只传 `text` 而缺少 `ref_audio_path` 等字段，代理会自动用配置文件中的默认角色补全。
也可在请求里传 `character`、`role`、`speaker` 等字段指定角色名。

### 命令行启动参数

```bash
python tts_proxy.py [--port 9882] [--config tts_gui_config.json]
```

## 6. 命令行调用（tts_reply.py）

### 基本用法

```bash
python tts_reply.py <角色名> <文本>
```

### 示例

```bash
python tts_reply.py 可莉 "今天天气真好！可莉想去原野抓松鼠，还要带上蹦蹦炸弹！"
```

### 在 Python 中调用

```python
import sys
sys.path.insert(0, r'G:\WorkBuddy\TTS加速器')
from tts_reply import tts_reply

tts_reply("可莉", "这是要朗读的内容，可以很长，程序会自动分句并行处理。")
```

## 7. 添加新角色音色

**方式一：通过 GUI（推荐）**

打开 GUI → 「角色管理」标签页 → 点「＋ 新增」，填写配置后「💾 保存角色」即可。

**方式二：直接编辑配置文件**

打开 `tts_gui_config.json`，在 `"roles"` 下添加：

```json
"新角色名": {
    "ref_audio_path": "TTS服务器上的参考音频路径",
    "prompt_text": "参考音频里说的文字内容",
    "prompt_lang": "zh",
    "text_lang": "zh",
    "text_split_method": "cut5",
    "speed_factor": 1.0
}
```

> ⚠️ `ref_audio_path` 是 TTS **服务器**上的路径，不是本机路径。

**语言代码说明：**

| 代码 | 语言 |
|------|------|
| `zh` | 中文 |
| `en` | 英文 |
| `ja` | 日语 |
| `yue` | 粤语 |
| `auto` | 自动检测 |

## 8. 服务器配置

### 当前默认服务器（示例）

配置了 **2 台物理机器**，每台多网卡冗余，任意一个 IP 可通即视为该机器在线：

```
机器1 - 首选
  http://192.168.28.3:9880/tts
  http://192.168.31.5:9880/tts
  http://192.168.28.7:9880/tts

机器2 - 备选
  http://192.168.31.177:9880/tts
  http://192.168.28.2:9880/tts
  http://192.168.28.127:9880/tts
```

### 修改服务器列表

通过 GUI「服务器管理」标签页增删改，或直接编辑 `tts_gui_config.json` 的 `server_groups` 字段：

```json
"server_groups": [
    {
        "name": "机器1（首选）",
        "enabled": true,
        "urls": [
            {"url": "http://192.168.x.x:9880/tts", "enabled": true}
        ]
    }
]
```

每个 `urls` 列表代表同一台物理机器的所有 IP，代理会从左到右尝试，通了就用。

## 9. 工作原理

### 智能分句

代理收到完整文本后，在本地切分再并发分发：

1. 按句末标点（`。！？…`）切分
2. 短于 8 字的碎片自动合并到下一句
3. 长于 50 字的句子按逗号/分号二次切分
4. 最终每句不超过 50 字

### 并发调度

所有句子同时并发，按全局 Round-Robin 偏移分配到不同服务器组：

- 多客户端同时请求时，各自的句子会错开分配到不同机器，充分利用所有服务器
- 单句请求也参与全局 RR，保证不同客户端的单句轮流分给不同机器

### 自动故障重试

某个 IP 不通 → 自动切换同台机器的其他 IP；整台机器失败 → 失败句子自动换另一台机器重试。全部服务器不可用时报 502，不会卡死。

### 音频合并

各句 WAV 片段按原始顺序无缝拼接，使用标准 `wave` 模块，**不依赖 FFmpeg 等外部工具**。

## 10. 调试模式

遇到异常响应（422、502 等）或文本处理问题时，可开启调试模式记录完整请求信息。

### 开启方式

GUI → 「代理服务」标签页 → 勾选「调试模式」

### 日志内容

日志写入 `tts_proxy_debug.log`，每次请求记录：

- 完整请求 payload（JSON，格式化输出）
- 关键字段速览（`text`、`text_lang`、`ref_audio_path`、`aux_ref_audio_paths` 等字段的类型和值）
- 请求阶段（解析后、角色补全后）

### 日志示例

```
============================================================
[14:23:11] 收到请求 (POST) 解析后
{
  "text": "今天天气真好！",
  "text_lang": "zh",
  ...
}
--- 关键字段 ---
  text: [str] '今天天气真好！'
  text_lang: [str] 'zh'
  aux_ref_audio_paths: [list] []
============================================================
```

## 11. 常见问题

### Q: 报 HTTP 422 错误

**原因：** 请求参数类型不匹配（最常见：`aux_ref_audio_paths` 传了空字符串而不是空列表）。

**代理服务已内置自动修正：** GET 参数传入的字符串会自动转换为正确类型（int/float/bool/list）。如果你直接调用 TTS 服务器而绕过代理，请检查 POST 的 JSON 字段类型。

---

### Q: `<br>` 或其他 HTML 标签被朗读出来

**原因：** 第三方客户端发送了含 HTML 标签的文本。

**代理服务已内置 HTML 清理：**
- `<br>` / `<br/>` → 转换为换行符
- 其他 HTML 标签（`<p>`、`<b>` 等）→ 直接移除
- HTML 实体（`&nbsp;`、`&amp;` 等）→ 还原为普通字符

如果还有问题，开启调试模式查看清理前后的 text 内容。

---

### Q: 提示「所有片段合成失败」

排查步骤：
1. `python check_tts_servers.py` 确认服务器在线
2. 检查防火墙是否阻止 9880 / 9882 端口
3. 确认本机和 TTS 服务器在同一局域网

---

### Q: 代理端口 9882 被占用无法启动

GUI 在「代理服务」标签页点「▶ 启动代理」时会自动清理占用该端口的旧进程。

手动清理：

```powershell
# 查找占用端口的 PID
netstat -ano | findstr ":9882"
# 结束进程（把 <PID> 替换为实际值）
taskkill /f /pid <PID>
```

---

### Q: 第一次运行很慢，之后快了

GPT-SoVITS 第一次推理需要加载模型到 GPU，属于正常现象。模型常驻显存后速度恢复正常。

---

### Q: 想修改语速

GUI → 「角色管理」→ 选中角色 → 拖动语速滑块 → 「💾 保存角色」。

或在 `tts_gui_config.json` 的角色配置里修改 `"speed_factor"`（1.0 = 正常，0.8 = 慢，1.3 = 快）。

---

### Q: 音频生成了但没有声音

- 检查系统音量和默认播放设备
- 在 `audio_dir` 目录找到对应 WAV 文件，手动双击测试能否播放
- Linux 上需要安装 `aplay`（`sudo apt install alsa-utils`）或 `paplay`

---

### Q: Windows 运行报编码错误（UnicodeEncodeError）

```powershell
$env:PYTHONIOENCODING="utf-8"
python tts_reply.py 可莉 "文本"
```
